The Document Object Model or DOM is a representation of the structure of your HTML
page you can interact with programmatically. The DOM's application programming interface (API) is exposed as
objects with properties and methods, enabling you to wright js code ot interact with teh HTML elements rendered on the page.

To work with the DOM you need to be able to know how to access it and obtain references to the elements you want to manipulate.

NOTE: The DOM is essentially a collection of nodes arranged in a tree. All the nodes are related to each other.
They are on big happy family of children, siblings, parents, grandparents, grandchildren, and so on. This essence of
a family tree represents the hierarchy of the DOM and is important to understand as you manipulate the DOM through code.

You can access DOM elements through a global object provided by the browser, called, document, or through
elements themselves after you obtain a reference to one.

Method Usage
getElementById Gets the individual element on the page by its unique id attribute value
getElementByClassName Gets all the elements that have the specified CSS class applied to them
getElementByTagName Gets all the elements of the page that have the specified tag name or element name
querySelector Gets the first child element found that matches the provided CSS selector criteria
querySelectorAll Gets all the child elements that match the provided CSS selector criteria
