article tag Defines sef-contained areas on the page
aside tag Defines smaller content areas outside the flow of a webpage
footer tag Defines the bottom of a section or page
header tag Defines the top of a section or page
hgroup tag Defines a group of headings (H1-H6 elements)
mark tag Defines text that should be highlighted
section tag Defines the distinct content of a document
nav tag Defines navigation to other pages in the site

Progress tag Defines the progress of the task



A Navigation tag (nav) usually have <a> tags which creates a hyperlink to other web pages. and will look like this.

<a href="http://www.google.com">Google</a>

Multiple links will look like this

Google Mozilla Web Docs w3schools


A Progress tag will have a value attribute specifying what the current value is and a max attribute to set the max possible value.

<span>0</span>

<progress value="21" max="100"></progress>

<span>100</span>

shows: →.... 0 100

(Note: using span will display it inline vs. with a paragraph tag which will indent)

Another example:

Data download is in progress, please wait patiently:

<progress max="5"></progress>

shows: →....



A div element is a way to structure your pages and style them correctly by using ids

<div id="pageHeader"></div>

<div id="LeftSide"></div>

<div id="RightSide"></div>

<div id="Footer"></div>

Using a div element however is like a basic building block where you can assign ids however if you use the
specific semantic elements then websites can fully leverage optimization for both search engines and screen readers
which will enable more accessibility and richer user experience.
