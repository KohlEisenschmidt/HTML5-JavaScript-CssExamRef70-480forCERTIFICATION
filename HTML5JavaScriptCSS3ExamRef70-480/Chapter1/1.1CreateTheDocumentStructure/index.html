Hypertext Markup Language   HTML   is make up of Tags/Elements to create the structure of a webpage

The following code demonstrates the basic template of and HTML5 page.

    <!DOCTYPE html>
    <html>
    <head>
       <meta charset="utf-8"/>
       <title> </title>
    </head>
    <body>

       <!-- page content goes here -->

    </body>
    </html>

These elements make up the core of HTML5

HTML5 element Description
<div> Defines self-contained areas on a page
<article> Defines a whole and complete composition or entry.
<aside> Defines smaller content area outside the flow of a webpage
<figcaption> Defines the caption of a figure element
<figure> Defines contents that contains a figure, such as an image, chart, or picture
<footer> Defines the bottom section or page
<header> Defines the top section of page
<hgroup> Defines a group of headings (H1-H6 elements)
<Mark> Defines text that should be highlighted
<nav> Defines navigation to other pages in the site
<progress> Defines the progress of a task
<section> Defines navigation to other pages in the site

Layout

It is important to give serious thought to the layout of your webpage so that your page is user friendly. If the entire page is styled as a single P element inside the body element, users might not come back due to them not being able to find what they are looking for.

The two most common methods to creating a layout in HTML5 using the Div or Table elements.

    
Div element layout method
   
        <div id="PageHeader"></div>
        <div id="LeftSide"></div>
        <div id="RightSide"></div>
        <div id="Footer"></div>
    
Table element layout method(Iv never seen used before)

        <table>
            <tr>
                <td colspan="3" id="Header"></td>
            </tr>
            <tr>
                <td rowspan="3" id="LeftBar"></td>
                <td rowspan="3" id="MainContent"></td>
                <div id="RightSideTop"></div>
            </tr>
            <tr>
                <div id="RightSideMiddle"></div>
            </tr>
            <tr>
                <div id="RightSideBottom"></div>
            </tr>
            <tr>
                <td colspan="3" id="Footer"></td> 
            </tr>
        </table>

Additionally using thead and tfoot elements can be used to provide a more semantic approach for labeling the cells. The concern using a table is its static nature of the structure.



Optimizing for search engines and screen readers

In the past (pre-HTML5) web designers predominantly used Div elements to structure a page. Today, there is a growing field for methodologies concerning layouts called Search Engine Optimization (SEO). When search engines search the internet they scour webpages for content so the user can receive relevant results. Search engines look through a web pages HTML and index content such as page and image metadata. There is a relationship then with the content on a website and how easily a user is able to find the website. With better descriptive elements using semantic markup a website can more easily be found by a user.

Another reason to carefully construct your layout using semantic markup is for screen readers. Screen-reader programs use text-to-speech algorithms that read aloud text on a page. This could be helpful for users who have difficulty viewing the webpage. Prior to HTML5 only the header elements (H1-H6) outlined a page and created a hierarchy for screen readers to use to present a table of contents to the user. 

        <h1>Fruits and Vegetables></h1>
        <h2>Fruits></h2>
        <h3>Apple></h3>
        <h3>Orange></h3>

With the use of semantic elements the document outline is changed and now each semantic element is considered a section that can house the original hierarchy header elements for screen readers. This intern will then make the webpage more defined and give more information to the Screen-reader for a better user experience. 

        <section>
            <h1>Fruits and Vegetables></h1>
            <section>
                <h1>Fruits></h1>
                <section>
                    <h1>Apple></h1>
                    <h1>Orange></h1>
